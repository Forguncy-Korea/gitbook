# 외부서비스 인증

포건시 서버관리자에서는 사용자가 비익명 서버 명령에 대한 접근 요청 시 Authorization Header에 OAuth2.0 접근 토큰을 추가하거나, 포건시 서버관리자의 User API에 접근할 수 있습니다.

서버사이드 명령어는 포건시의 핵심 기능 모듈로 출시 이후 좋은 반응을 얻고 있습니다. 포건시의 서버 단 명령 기능을 통해 개발자는 서버 단에서 실행되는 비즈니스 로직을 시각적으로 구성하고 프런트엔드와 백엔드의 분리를 실현할 수 있으며 이는 포건시의 고유한 장점을 완전히 반영합니다.&#x20;

포건시는 OAuth2의 "타사 애플리케이션 인증" 기능과 호환되므로 서버단 명령을 웹 API로 변환하고 호출을 위해 타사 시스템에 제공할 수 있습니다. "HTTP 요청 명령 보내기"와 함께 개발자는 포건시를 사용하여 프런트엔드 또는 백엔드 애플리케이션을 개발하고, 회사의 기존 시스템을 양방향으로 통합하고, 데이터 아일랜드를 개방하고, 다중 시스템 시너지 효과를 쉽게 달성할 수 있습니다.





### 외부 서비스 인증&#x20;

포건시 서버관리자에서 \[설정>웹보안설정]을 선택하여 웹 보안 설정 페이지로 들어갑니다.

<figure><img src="../../.gitbook/assets/image (1359).png" alt=""><figcaption></figcaption></figure>

타사 인증 영역에서 관리자는 각 타사에 대한 클라이언트 식별자 및 암호를 구성할 수 있습니다.

클라이언트 권한 추가를 클릭하여 클라이언트 권한 편집 페이지를 표시합니다.

<figure><img src="../../.gitbook/assets/image (1749).png" alt=""><figcaption></figcaption></figure>

| 설정         | 설명                                                                                                                                                                                                                                                    |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 클라이언트 이름   | 클라이언트 표시 이름(로깅용).                                                                                                                                                                                                                                     |
| 클라이언트 ID   | 클라이언트의 고유 ID입니다.                                                                                                                                                                                                                                      |
| 클라이언트 비밀번호 | 클라이언트 비밀번호                                                                                                                                                                                                                                            |
| 권한 부여 방법   | 인증 방법을 지정합니다. 암호, client\_credentials 또는 두 인증 방법의 조합을 지원합니다.                                                                                                                                                                                          |
| API 범위     | 접근 가능한 API 범위를 설정하고 ","를 사용하여 여러 범위를 구분합니다. 현재 " FGC\_UserApis  "는 사용자 추가 등과 같은 사용자 관리와 관련된 API에 대해 지원되며, 애플리케이션 이름(예: "재고 관리")을 정의된 서버 측 명령에 액세스하는 범위로 사용하는 API도 지원합니다 . 응용 프로그램에서. 모든 애플리케이션의 서버 명령을 찾으려면 "FGC\_AllAppsServerCommands"로 설정할 수 있습니다. |
| 토큰 유효 기한   | <p>액세스 토큰의 수명(초)(기본값은 7200초)이며 값은 0에서 99,999,999 사이의 정수로 설정할 수 있습니다.</p><p>새로 고침 토큰 지원: 기본적으로 선택되어 있지 않습니다. 선택하면 offline_access의 API 범위가 기본적으로 증가합니다. client_credentials 인증 방법은 새로 고침 토큰을 지원하지 않습니다.</p>                                              |
| 활성화 하기     | 확인 후 클라이언트가 활성화됩니다.                                                                                                                                                                                                                                   |

설정이 완료되면 설정 저장을 클릭합니다.
