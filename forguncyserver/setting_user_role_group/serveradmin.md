# 서버 관리자 개발과 서버 관리자

포건시에는 서버 관리자 개발과 서버 관리라는 두 가지 유형의 관리 플랫폼이 있습니다. 이 섹션에서는 차이점을 이해하는 데 도움이 됩니다.&#x20;

## 포건시 서버 관리자 개발&#x20;

리본메뉴에 \[보안-> 사용자 관리]를 선택하여 사용자 관리 플랫폼인 개발 시 사용자 계정 관리 플랫폼으로 이동합니다.

<figure><img src="../../.gitbook/assets/image (1893).png" alt=""><figcaption></figcaption></figure>

{% hint style="info" %}
기본 등록된 관리자 계정, 사용자 이름: Administrator, 암호: 123456 로그인합니다.
{% endhint %}

개발 시 사용자 계정 관리 플랫폼에서 만든 사용자, 구성된 조직 구조 및 메일 서버 정보는 응용 프로그램 테스트 및 개발에서만 사용할 수 있습니다.

앱이 공식적으로 온라인 상태가 되면 이 플랫폼을 사용하여 만든 사용자가 앱에 로그인하거나 현재 구성된 메일 서버를 사용하여 메시지를 보낼 수 없습니다.

## 포건시 서버 관리자&#x20;

포건시 서버에 들어가 포건시 서버관리자로 이동합니다.&#x20;

<figure><img src="../../.gitbook/assets/image (583).png" alt=""><figcaption></figcaption></figure>

{% hint style="info" %}
기본 등록된 관리자 계정, 사용자 이름: Administrator, 암호: 123456 로그인합니다.
{% endhint %}

서버관리자 페이지에서 만든 사용자, 설정된 사용자 정보 및 메일 서버 구성에서 앱이 온라인 상태가 되면 직접 사용할 수 있습니다.

## 서버관리자 개발 플랫폼은 서버관리자와 동일

1\. 둘 다 사용자, 역할 및 조직 구조를 만드는 데 사용할 수 있습니다.

2\. 둘 다 메일 서버, 암호 정책, 사용자 백업 및 복원을 설정할 수 있습니다.

3\. 둘 다 타사 보안 공급자를 구성할 수 있습니다.

## 서버 관리자 개발 플랫폼과 서버관리자의 차이점

1\. 색상이 다릅니다. 서버관리자 개발 플랫은 녹색이고 관리 콘솔은 파란색입니다.

2\. 라이선스 목록. 서버관리자 개발 플랫폼에는 라이선스 목록이 포함되지 않으며 관리 콘솔에는 관리 콘솔에서만 등록할 수 있는 라이선스 목록이 포함되어 있습니다.

3\. 응용 프로그램 관리. 백그라운드에서 모든 앱은 서버관리에서만 관리할 수 있습니다.
