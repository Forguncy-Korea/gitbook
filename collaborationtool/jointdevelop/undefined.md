# 권장사항 사용

이 섹션에서는 공동 개발을 위해 여러 사람이 공동 작업할 때 사용할 수 있는 몇 가지 권장 사항을 제공하여 보다 효율적이고 원활하게 공동 작업을 수행하고 앱을 만드는 데 도움이 됩니다.



* 공동 작업 엔지니어링 파일에서 플러그인을 사용하는 경우 모든 사용자가 동일한 버전의 플러그인을 로컬로 설치해야 하며 그렇지 않으면 오류가 발생합니다.
* 응용 프로그램 개발 초기에는 서로 다른 사용자가 서로 다른 부분을 담당하는 여러 부분으로 분할되므로 개발이 서로 영향을 미치지 않고 개발 효율성이 향상됩니다.
* 응용 프로그램 개발 전에 응용 프로그램의 인터페이스와 스타일을 디자인하고 버튼, 테이블 등과 같은 더 많은 요소를 사용하는 인터페이스를 디자인하여 개발자가 디자인 된 스타일을 직접 사용할 수 있으므로 나중에 요소의 스타일을 대규모로 조정하고 프로젝트의 진행을 늦출 수 있습니다.
* 데이터베이스는 모듈이며, 한 사용자가 편집할 때 다른 사용자가 사용할 수 없으며, 잘못된 피크 사용은 개발 효율성에 영향을 미치므로 데이터베이스를 미리 디자인하고 데이터베이스 유지 관리를 담당하는 사람이 사용하는 것이 가장 좋습니다.
