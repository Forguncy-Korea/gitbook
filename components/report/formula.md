# 식

보고서 식은 상수, 변수, 개체 멤버, 연산자, 함수 등과 같은 요소로 구성된 의미 있는 하위입니다.

일부 식을 자동으로 생성하는 것 외에도 보고서 내용 또는 형식 설정에 대한 사용정 식을 지원합니다.

## 표현식 이해하기&#x20;

먼저 식이 무엇인지 이해하는 데 도움이 되는 두 가지 간단한 예제를 사용합니다.

**예 1:** 텍스트 상자에 "{1+2}"를 입력하면 실제 미리 보기는 "3" 결과를 표시합니다.

![](<../../.gitbook/assets/image (1364).png>)

**예 2:** 텍스트 상자에 필드를 바인딩할 때(예: "범주 이름"을 바인딩할 때 텍스트 상자의 속성 설정 패널에서 실제 데이터가 자동으로 생성된 식 "{Count(범주 이름)}"임을 알 수 있습니다. 미리 볼 때 작업 후 결과 "10"이 표시됩니다.

![](<../../.gitbook/assets/image (482).png>)

또한 테이블 구성 요소의 테이블 행 번호와 보고서 모양의 조건부 서식 지정은 식로 원하는 결과를 얻을 수 있습니다.

이러한 예제를 통해 식은 등호 "="로 시작하고 미리 볼 때 계산되며 보고서 내용 표시 및 보고서 요소 동작 제어에 모두 사용할 수 있습니다.

## 일반적인 응용  프로그램 시나리오&#x20;

식의 구성 요소는 매우 풍부하므로 응용 프로그램은 광범위하고 유연합니다. 가장 일반적으로 사용되는 시나리오는 다음과 같습니다.

* **데이터 필드 값의 재처리입니다**

예를 들어 **{구매 수량 \* 제품 단가 \* (1 - 할인)}를** 사용하여 주문 금액을 얻습니다.

* **보고서 요소 동작 제어**

보고서 요소 표시 또는 숨기기(예: 동적 표시 구성 요소)

* **조건부 서식 지정**

프로세스 함수의 IIF 함수 예제와 같이 강조 표시된 주요 데이터

* **큐브 연결입니다**

예를 들어 Lookup 함수를 사용하여 다중 테이블 연결(예: 다른 함수의 Lookup 함수 예제)을 사용합니다.
