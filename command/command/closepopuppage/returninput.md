# 전달받을 값 변경 발생 시 상위 페이지의 입력란에 반환

팝업 페이지 닫기 명령을 설정하면 부모 페이지 바인딩 데이터 다시 로드로 설정된 항목이 있으며, 이 옵션을 선택하면 팝업 페이지를 닫고 상위 페이지로 돌아가면 페이지가 바인딩된 데이터를 다시 로드합니다.

일반적으로 상위 페이지에 테이블이 있으며 팝업 페이지가 업데이트되거나 새로운 데이터가 추가될 때 이 옵션을 체크하면 팝업 페이지를 닫은 후 상위 페이지의 테이블에 있는 데이터가 새로 고쳐집니다. , 페이지의 테이블에 있는 데이터가 최신 데이터가 되도록 합니다.

<figure><img src="../../../.gitbook/assets/image (2304).png" alt=""><figcaption></figcaption></figure>

* **"전달받을 값 변경 발생 시 상위 페이지의 입력란에 반환"을 선택하지 않을 때**&#x20;

예를 들어 주문 목록 페이지에서 추가 버튼을 클릭하여 페이지 추가를 팝업하고, 페이지 추가의 확인 확에서 데이터 테이블 작업 명령 뒤에 팝업 페이지 닫기 명령을 추가하고, 다음 그림과 같이 상위 페이지 다시 로드 바인딩된 데이터를 선택하지 않습니다.

![](<../../../.gitbook/assets/image (733).png>)

실행 후 주문 목록 페이지에서 데이터 추가 버튼을 클릭하고 페이지를 팝업하고 주문을 입력한 후 확인 단추를 클릭하면 방금 추가한 주문이 목록에 표시되지 않습니다.

![](<../../../.gitbook/assets/image (2212).png>)

F5 키를 누르거나 페이지 다시 로드를 클릭하면 방금 추가한 데이터가 표시됩니다.![](https://help.grapecity.com.cn/download/thumbnails/72353920/image2018-12-10_15-6-33.png?version=1\&modificationDate=1648092552000\&api=v2)

* **"전달받을 값 변경 발생 시 상위 페이지의 입력란에 반환"을 선택하지 않을 때**&#x20;

주문 정보 페이지에서 \[추가] 버튼을 클릭하여 \[페이지 추가]를 표시하고, \[페이지 추가]의 \[확인] 버튼에서 데이터 테이블 작업 명령 뒤에 \[팝업 페이지 닫기] 명령을 추가하고 다음 그림과 같이 \[전달받을 값 변경 발생 시 상위 페이지의 입력란에 반환]을 선택합니다.&#x20;

![](<../../../.gitbook/assets/image (2006).png>)

실행 후, 주문 목록 페이지에서, "데이터 추가"버튼을 클릭, 추가 페이지를 팝업, 주문을 입력 한 후 "추가"버튼을 클릭, 방금 추가 된 주문이 목록에 표시 된 것을 알 수 있습니다, 즉, 팝업 페이지를 닫는 동안 부모 페이지는 바인딩 된 데이터를 다시 로드, 새로 추가 된 데이터가 페이지에 표시 할 수 있도록 합니다.&#x20;

![](<../../../.gitbook/assets/image (2139).png>)
