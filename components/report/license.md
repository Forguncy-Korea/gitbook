# 보고서 라이선스

포건시 빌더에서 직접 보고서를 만들 수 있지만 앱이 게시된 후 보고서를 사용하려면 보고서 라이선스가 포함된 기본 서버 라이선스를 서비스에 미리 등록해야 합니다.

## 보고서 라이선스&#x20;

빌더에 보고서를 추가하면 보고서에 대한 사용 권한 정책을 묻는 프롬프트 상자가 나타납니다. 사용자가 이 정책에 동의하는 경우에만 보고서를 만들 수 있습니다.

보고서를 만들기 전에 체크박스를 선택하면 프롬프트가 더 이상 표시되지 않습니다.



보고서 라이센스는 기본 서버 라이센스에 포함되어 있습니다.

* 기본 서버 라이센스에 보고서 라이센스가 포함된 경우 다음과 같이 표시됩니다.

![](<../../.gitbook/assets/image (2121).png>)



서버에서 추가한 기본 서버 라이선스에 보고서 라이선스가 포함된 경우 사용자는 응용 프로그램을 게시한 후 만든 보고서를 사용할 수 있으며, 그렇지 않으면 빌더에서만 보고서를 사용할 수 있습니다.

보고서 라이센스가 포함된 기본 서버 라이센스가 서버에 추가되지 않은 경우:

* 보고서를 내보낼 때 PDF 문서에 워터마크가 있습니다.<br>
* 보고서를 열 때 사용자에게 보고서 라이선스를 등록하라는 메시지를 표시하는 보고서를 열 수 없습니다.
