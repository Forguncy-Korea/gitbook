# 서버단 명령

포건시는 서버단 명령을 정의한 후 페이지에 계산 논리를 배치하는 대신 서버 쪽에서 실행되는 서버 쪽 명령을 정의할 수 있도록 지원합니다. 이렇게 하면 동적 데이터 수집 및 계산의 논리를 더 잘 수행할 수 있습니다.

개발자는 시각적인 방식으로 서버 측에서 실행되는 비즈니스 로직을 개발하고 개발 작업 부하를 줄이면서 성능 최적화를 위한 더 많은 공간을 확보할 수 있습니다. 이 모든 것은 개발자가 더 큰 엔터프라이즈 수준 시스템의 문제를 쉽게 해결할 수 있도록 돕기 위한 것입니다.

서버측 명령을 사용하여 비즈니스 논리를 개발할 때 개발자는 빌더에서 테이블 지원에 의존하지 않는 명령을 만들 수 있으며 추상화된 비즈니스 처리 논리를 여러 페이지에서 직접 호출할 수 있습니다. PC, APP 등 멀티터미널 프로젝트를 개발하는 과정에서 이동식으로 제공되는 방식은 반복적인 작업을 획기적으로 줄여 개발 및 유지보수의 부담을 효과적으로 줄일 수 있습니다.

또한 최적화된 개발 방식으로 납기 단축 및 인건비 절감은 물론 시스템의 품질을 확보할 수 있습니다. 서버 측 명령은 중간 규모 및 대규모 프로젝트의 개발자가 팀 인력 비율 및 협업 방법을 최적화하는 데 도움이 될 수 있습니다. 예를 들어 기술 임계값이 낮고 사용자 상호 작용이 낮은 페이지 구축을 담당하는 특수 인력을 할당하고 복잡한 비즈니스를 위한 서버 측 명령을 지정할 수 있습니다. 경험이 많은 개발자에게 로직을 넘겨줄 수 있습니다. 두 가지가 동시에 작동하며 통합 후 더 빠른 속도로 프로젝트 납품을 완료할 수 있습니다.

서버 명령이 실행 중일 때 포건시는 서버에서 데이터 로드 및 계산을 수행한 다음 결과를 다시 클라이언트로 보냅니다. 기존 방법과 비교할 때 비즈니스 로직에서 사용하는 데이터 테이블은 클라이언트의 브라우저로 보낼 필요가 없습니다. 데이터 전송이 줄어들면 페이지 로딩 속도와 비즈니스 처리 성능이 크게 향상되어 최종 사용자에게 보다 원활한 경험을 제공할 수 있습니다. 기업 ERP, SCM 등 시스템 공통 재고 관리 모듈에서 재고 차감하는 시나리오를 예로 들면, 서버 측 명령어를 활용해 개발한 버전은 기존 방식에 비해 성능을 10배 이상 향상시킬 수 있다.&#x20;
