# 사용자 관리 명령 V2 (UserManagementCommand)

사용자, 역할, 조직관리, 사용자 정보 가져오기/내보내기, 관리사이트 URL 획득에 사용되는 5가지 사용자 관리 명령 세트가 제공됩니다.&#x20;

### 플러그인 다운로드&#x20;

버전에 맞는 플러그인을 다운로드 합니다.

### 1. 사용자 관리

<figure><img src="../../../.gitbook/assets/image (37).png" alt=""><figcaption></figcaption></figure>

* 로그인: 사용자를 로그인합니다. 비밀번호 확인 건너뛰기를 지원합니다.
* 로그아웃: 현재 사용자를 로그아웃합니다.
* 사용자 추가: 새로운 사용자를 추가합니다.
* 사용자 업데이트: 대상 사용자의 정보를 업데이트합니다.
* 사용자 삭제: 대상 사용자를 삭제합니다.
* 사용자 유효성 설정: 사용자가 유효한지 여부를 설정합니다. 유효한 매개변수: 0, 1. 그 중 0은 유효하지 않고 1은 유효합니다.
* 사용자 비밀번호 설정: 대상 사용자 비밀번호를 변경합니다.
* 역할에 사용자   추가: 사용자 그룹에 역할 세트를 일괄 할당합니다. 역할 ID는 사용자 관리 보기 생성 기능에서 역할 정보 보기에 추가할 수 있으며 역할 이름으로 검색할 수 있습니다.
* 역할에서  사용자 제거: 사용자 그룹과 역할 그룹을 일괄적으로 제거합니다.
* 사용자에  대한 역할 설정: 사용자 그룹의 역할을 일괄 설정합니다. 사용자의 원래 역할이 먼저 지워집니다.
* 조직에 사용자 추가: 사용자 그룹을 조직 노드에 일괄 추가합니다. 그 중 조직 노드 ID는 사용자 관리 보기 생성 기능에서 조직 정보 보기에 추가할 수 있으며, 조직 이름으로 검색할 수 있습니다.
* 조직에서 사용자 제거: 조직 노드 그룹에서 사용자 그룹을 일괄적으로 제거합니다.
* 사용자에   대한 조직 설정: 사용자 그룹의 조직 노드를 일괄 설정하려면 사용자의 원래 조직 노드가 먼저 지워집니다.
* 사용자에게 권한이 있는 역할 가져오기: 사용자에게 권한이 있는 모든 역할을 가져옵니다.

### 2. 역할관리

<figure><img src="../../../.gitbook/assets/image (38).png" alt=""><figcaption></figcaption></figure>

* 역할 추가: 새 역할을 생성합니다.
* 역할 업데이트: 대상 역할을 업데이트합니다.
* 역할 삭제: 대상 역할을 삭제합니다.
* 역할 폴더 추가: 새 역할 범주를 생성합니다.
* 역할 폴더 업데이트: 대상 역할 분류를 업데이트합니다.
* 역할 폴더 삭제: 역할 분류를 삭제합니다.
* 역할 또는 폴더 이동: 역할 및 역할 범주의 위치를 ​​조정하고 위, 아래 또는 아래로 이동합니다.
* 역할이 소유 권한 추가: 대상 역할의 경우 이 명령을 사용하여 다른 역할의 권한을 얻거나 자신이 가지고 있는 권한을 다른 역할에 부여합니다.
* 역할이 소유 권한 제거: 대상 역할의 경우 이 명령을 사용하여 다른 역할에서 기존 권한을 제거하거나 다른 역할에 할당된 권한을 제거합니다.
* 역할의 소유 권한 설정: 대상 역할의 경우 이 명령을 사용하여 어떤 역할의 권한을 설정하거나 어떤 역할에 권한을 할당합니다. 설정 전, 이전에 획득한 권한이 삭제되거나, 다른 역할에 할당된 권한이 삭제됩니다.
* 역할에 사용자 추가: 일괄적으로 역할에 사용자를 추가합니다.
* 역할에서 사용자 제거: 역할에서 사용자를 일괄적으로 제거합니다.
* 역할에 대한 사용자 설정: 사용자를 일괄적으로 역할에 설정합니다.
* 역할에 조직 추가: 조직 역할을 일괄적으로 추가합니다.
* 역할에서 조직 제거: 조직 역할을 일괄적으로 제거합니다.
* 역할에 대한 조직 설정: 조직 역할을 일괄적으로 설정합니다. 각 조직 노드의 대상 역할 설정이 먼저 지워집니다.



### 3. 조직관리

<figure><img src="../../../.gitbook/assets/image (39).png" alt=""><figcaption></figcaption></figure>

* 조직 추가: 새 조직 노드를 생성합니다.
* 조직 업데이트: 대상 조직 노드를 업데이트합니다.
* 조직 삭제: 대상 조직 노드를 삭제합니다.
* 조직 이동: 조직 노드의 위치를 ​​조정하여 상위, 동일, 하위로 이동합니다.
* 조직 구성원의 조직 역할 추가: 조직 구성원의 조직 역할을 일괄 추가합니다.
* 조직 구성원의 조직 역할 제거: 조직 구성원의 조직 역할을 일괄 제거합니다.
* 조직 구성원의 조직 역할 설정: 조직 구성원의 조직 역할을 일괄 설정합니다. 설정하기 전에 기존 조직 역할이 삭제됩니다.
* 조직 구성원의 리더 추가: 조직 내 대상 사용자를 일괄 감독자로 설정합니다. Zhang San을 재무 관리자로 설정하려면 먼저 \[조직에 사용자 추가]를 사용하여 재무 조직 노드에 Zhang San을 추가한 후 이 명령을 사용해야 합니다.
* 조직 구성원의 리더 제거: 조직 노드에 있는 감독자 사용자의 감독자 ID를 일괄 제거합니다(즉, 사용자-조직 관계 보기에서 감독자가 0으로 설정되어 있는지 여부). 이 명령은 감독자가 될지 여부만 제어하며 조직 노드에서 사용자를 제거하지는 않습니다.
* 조직 구성원의 리더 설정: 조직 노드에서 원래 감독자 ID를 모두 지운 후 대상을 감독자로 일괄 설정합니다. 즉, 사용자-조직 관계 보기에서 감독자가 1)인지 여부를 설정합니다.
* 조직에 사용자 추가: 조직 노드에 사용자를 일괄 추가합니다.
* 조직에서 사용자 제거: 조직 노드에서 사용자를 일괄 삭제합니다.
* 조직에    대한사용자 설정: 조직 노드에서 일괄적으로 사용자를 설정합니다.
* 역할에  조직 추가: 조직 역할을 일괄적으로 추가합니다. 역할 관리의 \[역할에 조직 추가]와 동일한 효과입니다.
* 역할에서 조직   제거: 조직 역할을 일괄적으로 제거합니다. 역할 관리의 \[역할에서 조직 제거]와 동일한 효과입니다.
* 조직에 대한 역할 설정: 조직 역할을 일괄적으로 설정합니다. 대상 조직 노드에 설정된 모든 역할이 먼저 지워집니다.



### 4. 가져오기 및 내보내기

<figure><img src="../../../.gitbook/assets/image (40).png" alt=""><figcaption></figcaption></figure>

* Excel에서 가져오기: Excel에서 시스템으로 사용자 정보를 가져옵니다. 증분 가져오기와 대체 가져오기의 두 가지 모드가 있습니다. 파일 이름은 서버에 이미 존재하는 파일 이름을 나타냅니다.
* Excel 내보내기: 사용자 정보를 Excel 형식으로 서버로 내보내려면 파일의 전체 경로(서버 경로 및 파일 이름 포함, 예: D:\temp.xlsx)를 설정해야 합니다.

<figure><img src="../../../.gitbook/assets/image (41).png" alt=""><figcaption></figcaption></figure>

일반적으로 RESTful API를 접속하는데 사용되는 관리 콘솔의 URL을 획득합니다.
