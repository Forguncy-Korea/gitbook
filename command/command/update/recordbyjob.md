# 고급 설정 - 작업별 레코드

데이터 테이블 작업 명령에서 작업 유형이 편집 또는 삭제인 경우 고급 설정에서 현재 행, 특정 행 또는 선택 행을 비롯한 작업별 레코드를 설정할 수 있습니다.

* 현재 행: 테이블의 현재 행은 포커스가 있는 행입니다. 현재 레코드는 테이블의 가시 범위 밖으로 스크롤되더라도 변경되지 않습니다. 페이지를 이동할 때 대상 페이지에 테이블의 데이터 원본 테이블에 바인딩된 셀이 있는 필드가 있는 경우 현재 행 정보도 대상 페이지로 전달됩니다.
* 특정 행: 조건을 충족하는 모든 행입니다.
* 리스트뷰의 선택한 행: 테이블의 행을 선택합니다. 행 선택은 행 헤더 또는 선택 열로 테이블에서 선택한 행을 나타냅니다.

![](<../../../.gitbook/assets/image (2213).png>)

## 현재 행 레코드 조작&#x20;

작업의 포커스가 있는 행의 레코드인 현재 행 레코드를 조작합니다.

예를 들어 데이터 테이블 작업 명령에서 작업 유형은 삭제이고 대상 테이블은 주문 테이블이며 현재 행을 선택합니다.

![](<../../../.gitbook/assets/image (551).png>)

실행 후 두 번째 행을 선택하고 삭제 단추를 클릭한 다음 확인을 클릭합니다. 주문 목록의 원래 두 번째 데이터 행인 A002가 삭제되고 현재 행이 주문 A003이 되었음을 알 수 있습니다.

![](<../../../.gitbook/assets/image (1423).png>)

## 특정 행 레코드 조작&#x20;

특정 행 레코드를 조작하려면 레코드를 필터링하는 조건을 설정해야 조건을 충족하는 레코드가 적절한 작업을 수행합니다.

예를 들어 데이터 테이블 작업 명령에서 작업 유형은 삭제되고 대상 테이블은 주문 테이블이며 특정 행을 선택하면 다음 그림과 같이 완료여부가 0인 드가 포함된 조건 설정 영역이 아래에 나타납니다.

![](<../../../.gitbook/assets/image (494).png>)

실행 후 삭제 버튼을 클릭하고 확인을 클릭합니다. 주문 목록의 완료여부가 0인  레코드가 모두 삭제되었음을 알 수 있습니다.

![](<../../../.gitbook/assets/image (2246).png>)

## 리스트뷰의 선택한 행&#x20;

행 레코드를 선택하는 작업은 페이지에서 테이블을 선택하여 테이블의 레코드를 선택하는 작업을 수행합니다. 행 선택은 열 선택으로 테이블에서 선택한 행을 나타냅니다.

예를 들어 데이터 테이블 작업 명령에서 작업 유형은 삭제이고 대상 테이블은 주문 테이블이며 행 선택을 선택하고 페이지에서 테이블을 선택해야 합니다.&#x20;

![](<../../../.gitbook/assets/image (2030).png>)

테이블은 다음 그림과 같이 열 선택을 선택합니다.

![](<../../../.gitbook/assets/image (1931).png>)

실행 후 주문 목록에서 처음 세 줄의 레코드 앞에 있는 체크박스 선택하고 삭제를 클릭한 다음 팝업 확인 대화 상자에서 확인을 선택하면 선택한 레코드의 처음 세 줄이 모두 삭제되었음을 알 수 있습니다.

![](<../../../.gitbook/assets/image (536).png>)
