# 데이터 명령 - 리스트뷰 업데이트 명령 만들기

리스트뷰 명령  작동을  사용하여 리스트뷰의 데이터에 대해 작업을 수행합니다. 리스트뷰에서 "편집", "추가"  "삭제"및 "삭제"가 활성화되지 않은 경우에도 이 명령을 통해 데이터를 업데이트할 수 있습니다.&#x20;

테이블 연산 명령에는 연산 행, 데이터, 선택, 연산 셀, 연산 열의 5가지 연산 유형이 있습니다.

<figure><img src="../../../.gitbook/assets/image (2066).png" alt=""><figcaption></figcaption></figure>

|       | 작업유형                                                                                        |
| ----- | ------------------------------------------------------------------------------------------- |
| 행 작동  | 편집, 추가, 삽입, 삭제                                                                              |
| 데이터   | 리스트뷰 반영, 데이터 새로 고침                                                                          |
| 선택    | 현재 행 번호 가져오기, 현재 열 번호 가져오기, 선택한 행 번호 목록 가져오기, 셀 선택, 선택한 모든 행 지우기, 모든 행 선택, 선택 행 추가, 행 선택 제거 |
| 셀 작동  | 값,가져오기,   텍스트 가져오기,  값설정, 텍스트 설정                                                            |
| 열 작동  | 열 숨기기, 열 보이기                                                                                |
| 상태정보  | 리스트뷰 상태 저장, 리스트뷰 상태 로드                                                                      |

## 데이터 저장 방식&#x20;

리스트뷰의 데이터 제출 모드에는 즉시 저장하기 모드와 즉시 저장해제 두 가지 유형이 있습니다. 기본 데이터 제출 모드는 즉시 저장이며 즉시 저장 해제를 하면 데이터가 즉시 저장되지 않습니다.&#x20;

#### 즉시 저장  <a href="#id-shu-ju-ming-ling-biao-ge-cao-zuo-jin-bang-ding" id="id-shu-ju-ming-ling-biao-ge-cao-zuo-jin-bang-ding"></a>

리스트에서 데이터를 수정하면 데이터가 실시간으로 데이터베이스에 커밋되고 이 패턴은 밀접하게 바인딩됩니다. 기본 테이블의 데이터 제출 모드는 즉시 저장 모드입니다.



#### 즉시 저장 해제  <a href="#id-shu-ju-ming-ling-biao-ge-cao-zuo-song-bang-ding" id="id-shu-ju-ming-ling-biao-ge-cao-zuo-song-bang-ding"></a>

즉시 저장 해제를 설정하면 새 레코드 추가, 기존 레코드 수정 또는 삭제를 포함하여 리스트뷰의 모든 작업이 즉시 데이터베이스에 업데이트되지 않습니다.



설정 방법은 리스트뷰를 선택하고 오른쪽 마우스 버튼을 클릭하여 리스트뷰 옵션 설정을 선택합니다.

리스트뷰 옵션 설정 대화상자에서 즉시 저장 해제를 체크합니다.
