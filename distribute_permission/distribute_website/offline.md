# 오프라인 배포

포건시 빌더는 웹 사이트를 포건시 서버에 직접 게시할 수 있습니다. 그러나 때로는 포건시빌더가 네트워크를 통해 포건시 서버에 직접 연결할 수 없으며 웹 사이트를 오프라인으로 게시 할 수 있습니다.

## 배포 가능한 파일을 저장&#x20;

만든 응용 프로그램을 포건시에서 직접 게시할 수 있는 파일 형식(.fgko)로 저장합니다.

빌더의 리본 메뉴 모음에서 배포-> 배포파일생성을 선택하여 파일을 로컬로 저장합니다.

![](<../../.gitbook/assets/image (1444).png>)

## 오프라인으로 웹사이트를 배포&#x20;

저장된 배포 가능한 파일(.fgko 파일)을 포건시 서버가 있는 컴퓨터에 복사하여 웹 사이트를 오프라인으로 게시할 수 있습니다.

포건시 서버의 설치 경로에서 "OfflinePublishTool .exe"을 찾아 두 번 클릭하여 설치 없이 게시 설정 창을 직접 엽니다.

<figure><img src="../../.gitbook/assets/image (132).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../../.gitbook/assets/image (133).png" alt=""><figcaption></figcaption></figure>

다음 설정을 설정한 후 확인을 클릭하여 사이트를 배포합니다.



| 파일 경로              | 오프라인 게시 파일(.fgccbs 파일)을 선택합니다.                                                                                                            |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------- |
| 응용 프로그램 이름         | 게시된 웹사이트의 이름입니다. 서버에 동일한 이름의 애플리케이션이 있는 경우 새로 게시된 애플리케이션이 이전 웹사이트를 덮어씁니다.                                                                 |
| 포트                 | <p>기본값은 80입니다.</p><p>포트 80이 다른 애플리케이션에 의해 점유되어 사용할 수 없는 경우 다른 포트 번호를 선택할 수 있습니다. 포트 번호의 값 범위는 1025에서 65534 사이입니다.</p>                     |
| HTTPS 웹사이트로 게시     | HTTPS 웹사이트로 게시해야 하는 경우에는 이동식 서버의 HTTPS 설정을 미리 구성해야 합니다. 자세한 내용은 HTTPS 게시 의 "HTTPS 설정"을 참조하세요.                                             |
| 데이터베이스 덮어쓰기        | <p>이미 게시된 웹사이트가 이동식 관리콘솔에 존재하는 경우, 이 옵션은 원본 웹사이트의 데이터베이스를 동시에 덮어쓸지 여부를 결정합니다.</p><p>웹 사이트를 처음 게시하는 경우 이 옵션을 선택했는지 여부에 관계없이 결과는 동일합니다.</p> |
| 서버 리소스 권한 재정의      | 선택하면 빌더에 설정된 리소스 권한이 서버에 게시되어 서버 측의 리소스 권한을 덮어씁니다. 자세한 내용은 리소스 권한을 참조하세요 .                                                                |
| 예약된 작업의 예약된 일정 재정의 | 선택하면 빌더에 설정된 예약된 작업이 서버에 게시되어 서버 측의 작업을 덮어씁니다.                                                                                            |
| 서버단 전역 변수 재정의      | 확인 후 빌더에서 설정한 전역 변수가 서버에 퍼블리시되어 서버 측의 전역 변수를 덮어쓰게 됩니다.                                                                                    |

## 오프라인으로 패키지를 배포하는 암호&#x20;

웹 사이트를 오프라인으로 배포할 때 서버가 게시자에게 암호를 제공하도록 오프라인 게시 파일의 암호를 설정합니다.

{% hint style="info" %}
&#x20;오프라인 게시 파일은 포건시빌더를 사용하지 않고 포건시 서비스 관리자에서 직접 게시할 수 있습니다. 따라서 암호를 설정한 응용 프로그램은 게시 후 3일 동안만 사용할 수 있으므로(게시자는 암호가 없음) 이러한 방식으로 평가판 웹 사이트를 게시할 수 있습니다.
{% endhint %}

파일->옵션-> 응용프로그램 설정을 선택하여 기타 영역에서 오프라인 배포 암호를 설정합니다.
