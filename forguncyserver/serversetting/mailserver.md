# 메일 서버 구성

메일 서버를 구성한 후 메시지를 보내 암호를 재설정하거나 데이터베이스 데이터 업데이트에 대한 메시지를 구독할 수 있습니다.

메일 서버는 포건시빌더와 서버관리자 모두에서 구성할 수 있습니다.

## 개발 테스트 단계&#x20;

빌더에서 \[보안-> 사용자 관리]를 선택하고 개발 시 사용자 계정 관리 플랫폼에서 SMTP(메일 서버 설정)를 선택합니다.

![](<../../.gitbook/assets/image (1763).png>)

## 정식 운영 단계&#x20;

공식 실행 단계에서 암호 재설정, 메일 명령 및 기타 메시지를 보낼 때 서버관리자에서 메일 서버를 구성해야 합니다.

서버관리자에서 설정-> 메일 서버 구성(SMTP)을 선택하여 메일 서버 설정 인터페이스를 입력합니다.

![](<../../.gitbook/assets/image (821).png>)

## 메일 서버 구성&#x20;

메일 서버의 구성은 아래 표에 나와 있습니다.

|           | 구성          | 설명                                      |
| --------- | ----------- | --------------------------------------- |
| 서버 세부 정보  | 보낸 사람 주소    | 표시할 보낸 사람의 전자 메일 주소를 지정합니다.             |
| 서버 세부 정보  | 보낸 사람 이름    | 표시할 보낸 사람의 이름을 지정합니다.                   |
| 서버 세부 정보  | 제목 접두사      | 보낼 메시지의 제목 접두사를 지정합니다.                  |
| 서버호스트     | 서버 호스트 이름   | 사용할 메일 서버의 서버 이름을 지정합니다.                |
| 서버호스트     | 서버 호스트 포트   | 사용할 메일 서버의 포트 번호를 지정합니다.                |
| 서버호스트     | 사용자 이름      | SMTP 서버에 인증이 필요한 경우 사용자 이름을 입력합니다.      |
| 서버호스트     | 암호          | SMTP 서버에 인증이 필요한 경우 암호를 입력합니다.          |
| 서버호스트     | SSL 활성화     | SMTP 서버가 SSL/TLS를 사용하여 보내고 확인하는지 여부입니다. |
| 테스트 이메일   | 테스트 이메일 주소  | 테스트 메시지를 받을 주소를 지정합니다.                  |
