# 데이터 테이블 편집하기

서버단 명령의 데이터 테이블 작업 명령에서 작업 유형을 업데이트로 설정하고 대상 테이블에서 지정된 레코드의 데이터를 업데이트할 수 있습니다.

다음은 서버 명령의 데이터 테이블 작업 명령에서 작업 유형이 편집될 때의 작업에 대해 설명합니다.

![](https://help.grapecity.com.cn/download/thumbnails/72357659/%E6%AD%A5%E9%AA%A41.png?version=1\&modificationDate=1648092609000\&api=v2)  개체 관리자의 서버단 명령 탭을 마우스 오른쪽 버튼을 클릭하고 서버단 명령 만들기를 선택하여 서버 단 명령 만들기 대화 상자를 나타납니다.

또는 폴더 만들기를 선택하여 폴더에 서버단 명령을 만듭니다.

![](<../../../../.gitbook/assets/image (466).png>)

리본 메뉴 모음에서 만들기를 클릭하고 서버 개체 영역에서 서버단 명령을 클릭하여 서버단 명령 만들기 대화 상자를 팝업할 수도 있습니다.

![](https://help.grapecity.com.cn/download/thumbnails/72357659/%E6%AD%A5%E9%AA%A42.png?version=1\&modificationDate=1648092609000\&api=v2)  서버 명령의 일반 설정을 편집합니다. 설정 명령의 이름은 데이터 테이블작업 입니다.

​![](https://help.grapecity.com.cn/download/thumbnails/72357613/%E6%AD%A5%E9%AA%A43.png?version=1\&modificationDate=1648092608000\&api=v2)서버단 명령을 편집하는 명령입니다. \[명령 편집] 하이퍼링크를 클릭하고 서버 명령 편집 대화 상자를 표시하고 데이터 테이블 업데이트하기 선택합니다.

* 업데이트 형식: 작업 유형을 추가로 선택합니다.
* 대상 테이블: \[고객주문테이블]과 같이 데이터를 추가할 데이터 테이블을 선택합니다.
*   필드 업데이트: 업데이트할 필드와 값을 설정합니다. 값은 고정 값, 수식 또는 파라미터일 수 있습니다(예: "고객 이름" 필드, "가나다" 값).

    업데이트된 필드 유형이 첨부 파일인 경우 업데이트된 값을 기존 파일로 설정하여 업데이트된 파일을 데이터베이스에 저장할 수 있습니다. 첨부 파일 필드에 여러 파일을 업데이트해야 하는 경우 | 사용하여 구분합니다.
* 조건 업데이트: 필드가 ID로 설정되고 작업이 같음, 값이 3, 즉 ID가 3인 데이터가 업데이트되는 경우와 같이 업데이트된 필드의 조건을 설정합니다.

![](<../../../../.gitbook/assets/image (1538).png>)

![](https://help.grapecity.com.cn/download/thumbnails/72357659/%E6%AD%A5%E9%AA%A44.png?version=1\&modificationDate=1648092609000\&api=v2)  설정완료되면 이 서버단 명령을 호출할 수 있습니다.

예를 들어 페이지에서 셀 범위를 선택하고 버튼으로 설정합니다. 버튼의 명령을 편집하고 "서버단 명령 호출"으로 명령을 선택한 다음 서버단 명령 뒤에 있는 드롭다운을 클릭하고 드롭다운 목록에서 "데이터 테이블작업" 선택합니다.

![](<../../../../.gitbook/assets/image (1732).png>)

![](https://help.grapecity.com.cn/download/thumbnails/72357659/%E6%AD%A5%E9%AA%A46.png?version=1\&modificationDate=1648092609000\&api=v2)  설정이 완료되면 \[확인]을 클릭하여 대화 상자를 닫습니다. 페이지를 실행합니다.페이지에서 데이터 테이블 업데이트 버튼 클릭하면 서비에서 데이터 테이블 업데이트가 수행됩니다.

ID 3 레코드에서 고객 이름이 전현무에서 가나다로 업데이트된 것을 볼 수 있습니다.

![](<../../../../.gitbook/assets/image (1593).png>)
