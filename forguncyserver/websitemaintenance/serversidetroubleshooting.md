# 서버단 문제 해결

포건시 빌더(디자이너)를 통해 서버에 배포된 응용 프로그램이 제대로 작동하지 않으면 다음을 확인합니다. &#x20;

## 서버 확인&#x20;

**항목 1. 포건시 서버가 설치된 컴퓨터가 제대로 작동하는지 확인합니다.**

컴퓨터 또는 내 컴퓨터에서 마우스 오른쪽 버튼 클릭하고 관리를 선택합니다.

<figure><img src="../../.gitbook/assets/image (1661).png" alt=""><figcaption></figcaption></figure>

팝업 컴퓨터 관리자에서 서비스를 선택하고 서비스 목록에서 Forguncy Server Service를 찾아 상태가 시작됨 또는 실행 중인지, 시작 유형이 자동인지 확인합니다.

그렇지 않은 경우 시작 유형을 자동으로 설정하고 서비스를 다시 시작합니다.<br>

<figure><img src="../../.gitbook/assets/image (1626).png" alt=""><figcaption></figcaption></figure>



**항목 2. 포건시가 설치된 컴퓨터의 설정을 확인합니다.**

포건 응용 프로그램은 Ajax를 통해 실행 중인 서버와 통신하기 위해 페이지 콘텐츠를 읽음으로써 응용 프로그램이 상대 경로에서 이루어지는 단일 페이지 응용 프로그램(SPA)입니다.

포건 서버가 설치된 컴퓨터가 역방향 프록시 서버를 설정하고 요청 전송 중에 도메인 이름과 역방향 프록시가 하위 디렉터리 정렬을 설정한 경우 활자 그리드가 제대로 작동하지 않을 수 있습니다.

**항목 3. 외부 데이터베이스 연결을 확인합니다.**

외부 데이터베이스가 포건 서버에 게시된 후 제대로 표시되지 않으며 외부 데이터베이스 오류일 수 있으므로 [외부 데이터베이스에 연결](../../database/externaldb/)하려면  참조하십시오.

**항목 4.  서버 측 컴퓨터 이름과 IP 주소를 확인합니다.**

포건 서버는 응용 프로그램이 게시될 때 서버 이름/IP 주소인 서버 이름/IP 주소를 사용하여 사용자 계정을 관리하는 페이지에 액세스합니다. 따라서 서버와 IP 주소 간의 대응이 제한되거나 보안이 설정된 경우 응용 프로그램이 제대로 작동하지 않습니다.

해결 방법은 다음 두 가지 방법을 참조하십시오.

1. 호스트 파일에서 서버 이름이 로컬 IP 주소와 일치하는지 확인합니다.
2. 지정된 로컬 IP 주소를 사용하여 앱 배포를 수행합니다.
