# 서버단 명령을 디버그

비즈니스 요구 사항 및 사용 시나리오에 따라 매우 복잡할 수 있는 포건시에서 서버 명령을 만들고 일부 오류가 발생하면 원인을 찾기가 어려울 수 있습니다.

포건시는 서버단 명령 실행 중에 로그 기능을 제공하며 로그를 검사하여 서버 쪽 명령을 디버깅하는 데 도움을 줄 수 있습니다.

디버깅을 위해 Google 크롬을 사용하는 것이 좋습니다. 디버그 도구는 Google 크롬 버전에 따라 다를 수 있습니다.

## 서버단 명령을 디버그하기&#x20;

서버단 명령을 만든 후 페이지 측에서 서버단 명령을 호출할 수 있습니다. 서버단 명령이 요청을 전송하여 실행을 트리거하면 브라우저의 콘솔에 로그 정보가 자동으로 표시됩니다. 브라우저에서 F12 키를 눌러 개발자 도구를 열어 콘솔의 정보를 볼 수 있습니다.

예를 들어 다음 그림과 같이 매개 변수와 명령이 있는 보다 복잡한 서버단 명령을 만듭니다.

<figure><img src="../../.gitbook/assets/image (443).png" alt=""><figcaption></figcaption></figure>

페이지에서 호출 서버 명령을 설정한 다음 페이지를 실행하고 페이지의 주문 세부 정보 테이블 테이블에서 데이터를 추가, 수정 및 삭제한 다음 주문 세부 정보 업데이트 버튼을 클릭하면 서비스 쪽에서 순환 명령의 데이터 테이블 작업 명령을 실행합니다.

브라우저에서 F12 키를 눌러 개발자 도구를 열고 콘솔을 클릭하면 실행 중인 서버 측 명령의 이름, 요청 방법, 사용자, 매개 변수 및 명령을 포함하여 서버가 실행한 후 로그 정보를 볼 수 있습니다.

서버단 명령이 오류로 실행되는 경우 여기에 있는 정보를 기반으로 디버깅을 수정할 수 있습니다.
