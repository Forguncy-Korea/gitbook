# 리소스 권한&#x20;

애플리케이션의 다양한 리소스에 대한 권한은 빌더에서 설정되며, 애플리케이션을 게시한 후 페이지 권한, 셀 권한, 서버 명령 권한, 서버 알림 권한, 보고서 권한, 프로세스 권한 등을 서버관리자에서 보고 관리할 수 있습니다.&#x20;

서버관리자 진입 후 "허가"을 선택하고 이동형 애플리케이션 목록에서 애플리케이션을 선택한 후 "리소스 권한"을 클릭합니다 .

리소스 권한을 설정한 후 설정 저장을 클릭하면 권한이 적용됩니다.

<figure><img src="../../.gitbook/assets/image (2270).png" alt=""><figcaption></figcaption></figure>

### 역할별 권한 설정&#x20;

"역할별 권한 설정" 모드를 선택하면 현재 애플리케이션의 모든 역할이 나열됩니다. 역할을 선택하면 현재 역할이 소유한 다양한 리소스의 권한이 페이지 오른쪽에 나열됩니다.

탭을 전환하여 역할의 리소스 권한을 보고 리소스 앞의 확인란을 선택하거나 선택 취소할 수 있습니다.

"이 역할의 기존 권한을 보려면 클릭하십시오"를 클릭하면 팝업 카드에서 이 역할의 기존 권한을 볼 수 있습니다.

<figure><img src="../../.gitbook/assets/image (1951).png" alt=""><figcaption></figcaption></figure>

### 모듈별 권한 설정&#x20;

"모듈별 권한 설정" 모드를 선택하면 현재 애플리케이션의 모든 리소스 권한이 나열됩니다. 리소스 권한을 선택하면 현재 리소스에 설정된 권한이 있는 모듈 및 역할이 페이지에 나열됩니다.

탭을 전환하여 리소스 권한을 볼 수 있으며 역할 앞의 확인란을 선택하거나 선택 취소할 수 있습니다.
