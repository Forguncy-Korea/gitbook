# 앱 배포 내역

애플리케이션 게시 시 "프로젝트 파일을 서버에 게시"를 체크하면 서버에 애플리케이션을 게시한 후 포건시 서버 관리자 에서 해당 애플리케이션의 이력을 확인하고 관리할 수 있습니다



포건시서버관리자  에서 앱 목록을 클릭하고 앱을 선택한 후 앱 이을 선택하세요.

<figure><img src="../../.gitbook/assets/image (141).png" alt=""><figcaption></figcaption></figure>

### 설정 <a href="#she-zhi" id="she-zhi"></a>

보관할 최대 프로젝트 파일 수를 설정할 수 있으며 기본값은 20입니다. 최대 개수는 100개까지 설정할 수 있습니다.

프로젝트 파일은 시간의 내림차순으로 정렬됩니다. 파일 개수가 설정된 최대 개수를 초과하면 가장 오래된 프로젝트 파일부터 삭제됩니다.

설정이 완료되면 "설정 저장" 버튼을 클릭하세요.



### 애플리케이션 작업 로그 <a href="#ying-yong-cao-zuo-ri-zhi" id="ying-yong-cao-zuo-ri-zhi"></a>

"포건시서버에서 이 애플리케이션을 실행하기 위한 로그 보기"를 클릭하면 다음 대화 상자가 나타납니다.

<figure><img src="../../.gitbook/assets/image (142).png" alt=""><figcaption></figcaption></figure>

감사 로그 대화 상자에서 기본 검색 조건에는 애플리케이션, 로그 유형, 작업 애플리케이션 이름이 포함됩니다.



### 애플리케이션 파일 기록 <a href="#ying-yong-wen-jian-li-shi" id="ying-yong-wen-jian-li-shi"></a>

애플리케이션 파일 목록에는 서버에 공개된 모든 프로젝트 파일의 이력이 나열됩니다.

시간, 프로젝트 파일 이름 및 파일 크기를 포함합니다. 파일을 다운로드하거나 삭제할 수 있습니다.
