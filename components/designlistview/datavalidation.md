# 데이터 유효성 검사 기간

포건시는 데이터 작업 명령이 실행될 때와 테이블 형식 데이터가 제출될 때를 포함하여 데이터 유효성 검사 타이밍을 설정할 수 있도록 지원합니다.

* 데이터 변경 명령 실행: Excel, CSV를 가져오거나 테이블에 데이터를 붙여 넣는 등의 작업이 수행되면 테이블 데이터 검사가 즉시 트리거됩니다.
* 리스트뷰 반영 : Excel, CSV를 가져오거나 테이블에 데이터를 붙여 넣는 등의 작업은 테이블 데이터 유효성 검사를 즉시 트리거하지 않으며 테이블 데이터를 제출할 때만 트리거됩니다.



## 데이터 유효성 검사

데이터 유효성 검사를 설정하려면 리스트뷰가 편집 허용되고 즉시 저장 해제 선택해야 합니다.

리스트뷰를 선택하고 마우스 오른쪽 버튼을 클릭한 다음 마우스 오른쪽 버튼 클릭 메뉴에서 리스트뷰 옵션설정을 선택하여 리스트뷰 설정 대화 상자를 표시합니다.

편집 선택 탭에서 편집 허용을 선택합니다.

![](<../../.gitbook/assets/image (1204).png>)

편집 허용을 설정하고 나서, 데이터 탭에서 즉시 저장 해제를 선택합니다. 이 옵션을 선택하면 레코드 추가, 기존 레코드 수정 또는 삭제를 포함하여 페이지 테이블의 모든 작업이 즉시 데이터베이스에 업데이트되지 않습니다.

![](<../../.gitbook/assets/image (1202).png>)

이 시점에서 데이터 작업 명령이 실행될 때와 리스트뷰 데이터가 제출될 때를 포함하여 데이터 유효성 검사 타이밍을 설정할 수 있습니다.

* 데이터 작업 명령 실행: Excel, CSV를 가져오거나 테이블에 데이터를 붙여 넣는 등의 작업이 수행되면 테이블 데이터 검사가 즉시 트리거됩니다.
* 테이블 데이터 제출: Excel, CSV를 가져오거나 테이블에 데이터를 붙여 넣는 등의 작업은 테이블 데이터 유효성 검사를 즉시 트리거하지 않으며 테이블 데이터를 제출할 때만 트리거됩니다.

예를 들어 주문 목록 페이지에서 데이터 제출 버튼을 설정하고 편집 명령을 리스트뷰 작업 명령으로 설정합니다. 작업 유형은 제출 양식입니다.

![](<../../.gitbook/assets/image (1503).png>)

#### 데이터 작업 명령이 실행될 때&#x20;

![](<../../.gitbook/assets/image (1557).png>)

CSV 파일의 데이터를 복사하여 리스트뷰에 붙여넣으면 리스트 데이터 검사가 즉시 트리거됩니다.

![](<../../.gitbook/assets/image (568).png>)

#### 리스트뷰 데이터가 제출될 때

리스트뷰 켜서 새 행을 편집하고 추가할 수 있도록 허용한 리스트뷰 반영을 선택하고 데이터 유효성 검사 타이밍을 선택하여 테이블 데이터 제출 시 페이지를 실행합니다.

![](<../../.gitbook/assets/image (1053).png>)
