# 승인 프로세스

프로세스에 승인을 위해 여러 역할이 필요한 경우 여러 단계를 추가하는 일반 프로세스 대신 승인 프로세스를 설정할 수 있습니다. 또한 경우에 따라 승인자는 동일한 레코드를 여러 번 승인해야 합니다(예: 프로젝트 관리자-부서 관리자-일반 관리자"의 순서로 응용 프로그램을 승인해야 하며 프로젝트 관리자가 제출한 응용 프로그램의 경우 이 프로세스를 따라야 하는 경우 응용 프로그램을 제출하고 승인해야 합니다.)&#x20;

부서 관리자가 제출한 요청인 경우 승인을 위해 일반 관리자에게 전달하기 전에 두 번 제출하고 승인해야 합니다. 이 경우 승인 프로세스를 사용하여 프로세스를 간소화할 수 있습니다.

승인 프로세스 중에 신청자의 역할이 승인 프로세스에 있는 경우 해당 사람이 제출한 응용 프로그램은 다음 승인 단계로 직접 전송됩니다. 연속된 두 개 이상의 승인 링크의 승인자가 동일한 경우 한 번만 승인하면 다음 승인 단계로 이동할 수 있습니다.&#x20;

예를 들어, 신청서는 프로젝트 관리자-부서 관리자-일반 관리자의 순서로 승인되어야 하며, 프로젝트 관리자가 제출한 경우 제출 후 부서 관리자의 승인을 받아야 합니다. 부서 관리자가 제출한 신청서의 경우, 제출 후 일반 관리자의 승인을 위해 직접 입력됩니다. 프로젝트 관리자와 부서 관리자가 동일한 사람인 경우 일반 직원이 제출한 신청서는 부서 관리자의 승인을 한 번만 거치면 일반 관리자의 승인을 받을 수 있습니다.

### 승인 프로세스 추가&#x20;

프로세스 단계를 선택하고 승인 프로세스 추가를 클릭하면 프로세스 단계 뒤에 승인 프로세스를 추가할 수 있습니다.

### 승인 프로세스 설정&#x20;

승인 프로세스 단계를 선택하여 원래 상태, 승인 프로세스, 조건, 대상 상태 및 할당대상을 아래에 설정합니다.

승인 절차 하이퍼링크를 클릭하여 승인 프로세스 설정 페이지로 이동합니다.

<figure><img src="../../.gitbook/assets/image (896).png" alt=""><figcaption></figcaption></figure>

승인 프로세스 설정에 대한 설명은 아래 표에 나와있습니다.&#x20;



| 설정                        | 설명                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 신청자                       | 승인 프로세스에서 처리하는 레코드의 요청자입니다. 작성자 또는 워크플로우를 열도록 선택할 수 있는 테이블 유형은 사용자의 필드입니다.                                                                                                                                                                                                                                                                                |
| 승인 링크를 추가/삭제/위로 이동/아래로 이동 | 승인 단계를 추가/제거하고 위쪽 및 아래쪽 화살표를 클릭하여 승인 프로세스의 순서를 이동할 수 있습니다.                                                                                                                                                                                                                                                                                                |
| 승인자                       | 승인 세션에 기록된 프로세서입니다. 승인자의 역할을 선택합니다. 역할 설정은 사용자 관리에서 수행되며 [역할 만들기](../../forguncyserver/setting_user_role_group/createrole.md)를 참조하십시오.                                                                                                                                                                                                                    |
| 조직 기반 필터링                 | 신청자가 있는 조직 노드를 기준으로 승인자를 필터링할지 여부입니다. 조직의 구조 및 수준 설정은 [조직 구조 만들기](../../forguncyserver/setting_user_role_group/)를 참조하십시오.                                                                                                                                                                                                                                 |
| 승인 입력 조건                  | 신청자가 제출한 레코드가 이 승인 단계로 이동하는 데 필요한 조건입니다.                                                                                                                                                                                                                                                                                                                  |
| 승인 명령 설정                  | 승인 프로세스 전송 전후, 거부 프로세스 전송 전후의 명령을 설정합니다.                                                                                                                                                                                                                                                                                                                  |
| 다중 승인 정책                  | <p>승인 프로세스의 승인자가 여러 개인 경우 승인자 승인, 임의 선택 승인, 모든 승인자 승인을 선택하도록 설정할 수 있습니다.</p><ul><li>승인자 선택: 승인자 목록에서 레코드 중 하나를 선택하여 레코드를 승인합니다.</li><li>아무나 승인: 승인자 목록의 모든 사람이 레코드를 승인한 후 링크가 종료되고 다음 단계로 이동합니다. 일반 프로세스에서 "최소 1명 승인 시 다음으로 이동" 효과와 동일합니다.</li><li>모든  승인: 승인자 목록의 모든 사용자가 레코드를 승인한 후에만 링크가 종료되고 다음 단계로 이동합니다. 일반 프로세스의 "모두 승인 다음으로 이동"과 동일합니다.</li></ul> |
| 정책 거절                     | <p>승인이 거부된 경우 원래 상태로 돌아가기, 단계별 반품, 프로세스 종료로 설정할 수 있습니다.</p><ul><li>처음으로 돌아가기: 승인상태를 원래 상태로 돌아갑니다.</li><li>이전으로 돌아가기: 승인상태를 단계별로 아갑니다.</li><li>프로세스 종료: 구매요청을 직접 마감하고 프로세스를 종료합니다.</li></ul>                                                                                                                                                               |
| 승인 작업 이름 / 거절된 작업 이름      | 승인/거부 버튼에 표시되는 텍스트를 사용자 지정할 수 있습니다.                                                                                                                                                                                                                                                                                                                       |
